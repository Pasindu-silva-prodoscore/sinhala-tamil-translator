# Models Directory — Agent Guide

## Structure

```
models/
├── baseline/
│   ├── model.pt              ← PyTorch state dict for the direct Si→Ta Transformer
│   ├── meta.json             ← Vocabulary sizes, hyperparameters, training loss history
│   └── hypothesis.ta         ← Model translations of the test set (used in evaluate.py)
│
└── cascade/
    ├── intermediate_en.txt   ← Step 1 output: Sinhala → English translations
    └── final_ta.txt          ← Step 2 output: English → Tamil (final cascade hypothesis)
```

## Loading the Baseline Model

```python
import torch
from src.train_baseline import BaselineTransformer, build_vocab_from_spm_vocab

si_vocab = build_vocab_from_spm_vocab('data/processed/spm/si_spm.model')
ta_vocab = build_vocab_from_spm_vocab('data/processed/spm/ta_spm.model')

model = BaselineTransformer(
    src_vocab_size=len(si_vocab),
    tgt_vocab_size=len(ta_vocab),
    d_model=512
)
model.load_state_dict(torch.load('models/baseline/model.pt', map_location='cpu'))
model.eval()
```

## Intermediate English Files

`cascade/intermediate_en.txt` is generated by Step 1 of the cascade (Si→En).  
It is kept separately for **error propagation analysis**: by comparing these English outputs
to the final Tamil outputs, you can see how translation errors from Step 1 compound in Step 2.

This is directly relevant to Research Sub-question 2:
> "How does error propagation through the English pivot stage affect overall translation quality?"

## Checkpointing During Full Training

For the full research system training on Kaggle (30hr session limit), save checkpoints every 1,000 steps:

```python
if step % 1000 == 0:
    torch.save(model.state_dict(), f'models/baseline/checkpoint_step{step}.pt')
```

Resume from the latest checkpoint by loading state dict before continuing training.
