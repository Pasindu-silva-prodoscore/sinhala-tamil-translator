# Sinhala-Tamil Pivot MT — Proof of Concept

> Research project: using English as a pivot language to overcome the Sinhala-Tamil parallel data scarcity barrier.

## Background

Direct Sinhala-Tamil machine translation is stuck at BLEU 21.26 (Nissanka et al. 2020) — below the BLEU 30 threshold for practical usefulness — because only ~25,000 training sentence pairs exist. This PoC tests the hypothesis from Pushpananda et al. (2018): routing translation through English as a pivot language should improve quality by leveraging the much larger English-Sinhala and English-Tamil corpora.

## What This PoC Demonstrates

```
Si → En → Ta   (cascade pivot, using pretrained models)
    vs
Si → Ta        (direct Transformer, trained on small corpus)
```

The cascade substantially outperforms the direct model, proving the pipeline works before full-scale training.

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Generate sample data (already done — 118 pairs in data/raw/)

```bash
python data/generate_sample_data.py
```

### 3. Run the full pipeline in the notebook

```bash
jupyter notebook poc.ipynb
```

Run all cells top to bottom. The notebook walks through every phase with explanations and produces a comparison report in `results/`.

### 4. Run individual phases from the command line

```bash
# Phase 1 — preprocess
python src/preprocess.py \
  --si data/raw/corpus.si \
  --ta data/raw/corpus.ta \
  --out data/processed \
  --test-out data/test_set \
  --test-size 20

# Phase 1b — LaBSE filter
python src/labse_filter.py \
  --si data/processed/train.si \
  --ta data/processed/train.ta \
  --threshold 0.85

# Phase 2 — tokenise
python src/tokenise.py train \
  --si data/processed/train_filtered.si \
  --ta data/processed/train_filtered.ta \
  --model-dir data/processed/spm \
  --vocab-size 4000

python src/tokenise.py apply \
  --si data/processed/train_filtered.si \
  --ta data/processed/train_filtered.ta \
  --model-dir data/processed/spm

# Phase 3 — train baseline
python src/train_baseline.py \
  --train-si data/processed/tokenised/train_filtered.si \
  --train-ta data/processed/tokenised/train_filtered.ta \
  --si-spm data/processed/spm/si_spm.model \
  --ta-spm data/processed/spm/ta_spm.model \
  --epochs 20

# Phase 4 — cascade pivot
python src/cascade_pivot.py \
  --si data/test_set/test.si \
  --out models/cascade

# Phase 5 — evaluate
python src/evaluate.py \
  --reference data/test_set/test.ta \
  --baseline  models/baseline/hypothesis.ta \
  --cascade   models/cascade/final_ta.txt \
  --source    data/test_set/test.si \
  --cascade-en models/cascade/intermediate_en.txt
```

For IndicTrans2 on Kaggle: use a P100 notebook (free tier, 16GB VRAM).

## Expected Results (PoC Scale)

| System | BLEU | chrF |
|--------|------|------|
| Baseline (direct, ~100 pairs, 20 epochs) | 3–8 | 15–25 |
| Cascade (pretrained Si→En→Ta) | 12–22 | 30–45 |

The cascade wins by a large margin with **no fine-tuning** on the target domain, purely from the pretrained models. Fine-tuning (done in the full research system) will push these scores higher.

## Full Research System Differences

This PoC uses ~118 pairs and pretrained models. The full research system (dissertation):

| Component | PoC | Full System |
|-----------|-----|-------------|
| Training data | 118 pairs | ~25,000 (direct) + millions (bridge) |
| Si→En model | Helsinki-NLP (no fine-tune) | Fine-tuned on government corpus |
| En→Ta model | IndicTrans2 (no fine-tune) | Fine-tuned on government Tamil-English |
| Parameter freezing | Demonstrated | Applied in training |
| Test set | 20 sentences | 500 sentences (published) |
| Human evaluation | None | 200 sentences, 3 bilingual raters |
| Vocab size | 4,000 | 8,000 |
